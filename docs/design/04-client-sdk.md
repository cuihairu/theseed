# Client SDK Design — 客户端引擎支持

> theseed 客户端 SDK 设计，覆盖 Unity、Unreal Engine 5、Cocos Creator 三大游戏引擎。
> 核心原则：**一份协议定义 → 自动生成各引擎的客户端代码**，开发者不需要手写同步逻辑。

---

## 1. 架构总览

```
┌──────────────────────────────────────────────────────────────────┐
│  Def 文件 (YAML)                                                 │
│  "一份定义，四处生成"                                             │
├──────────────────────────────────────────────────────────────────┤
│  Code Generator (theseed-codegen)                                │
│  ├── Server C++ Entity 类                                        │
│  ├── Client C# (Unity) 类                                        │
│  ├── Client C++ (UE5) 类                                         │
│  └── Client TypeScript (Cocos) 类                                │
├──────────────────────────────────────────────────────────────────┤
│  Runtime SDK (每个引擎的运行时库)                                 │
│  ├── Network Layer (TCP / WebSocket)                             │
│  ├── Entity Sync Engine (属性同步、插值、预测)                    │
│  ├── Event System (消息收发)                                      │
│  └── Debug Tools (编辑器内嵌)                                    │
└──────────────────────────────────────────────────────────────────┘
```

## 2. 协议生成器 (theseed-codegen)

### 2.1 输入：YAML 定义

```yaml
# defs/entities/Avatar.def.yaml
entity Avatar:
  sides: [base, cell]

  properties:
    name:
      type: STRING
      size: 64
      flags: [base, cell, client, persistent]

    level:
      type: UINT32
      default: 1
      flags: [base, cell, client, persistent]

    hp:
      type: FLOAT32
      default: 100.0
      flags: [cell, client]
      detail_level: 0              # detail_level=0: 高精度，始终同步
      interpolate: true            # 客户端插值
      interpolate_speed: 5.0       # 插值速度

    position:
      type: VECTOR3
      flags: [cell, client]
      detail_level: 1              # detail_level=1: 低精度，远距离同步
      interpolate: true
      interpolate_speed: 10.0      # 位置插值速度（单位/秒）

    rotation:
      type: VECTOR3
      flags: [cell, client]
      detail_level: 1
      interpolate: true

    mp:
      type: FLOAT32
      default: 50.0
      flags: [cell, client]
      interpolate: false           # 不插值，直接跳变

  methods:
    # 客户端可调用（exposed）
    attack:
      side: cell
      args: [targetId: ENTITY_ID, skillId: UINT32]
      exposed: true

    useItem:
      side: base
      args: [itemId: UINT32, count: UINT32]
      exposed: true

    move:
      side: cell
      args: [position: VECTOR3, rotation: VECTOR3]
      exposed: true

    # 服务器推送到客户端
    onLevelUp:
      side: client
      args: [newLevel: UINT32]

    onDamaged:
      side: client
      args: [damage: FLOAT32, attackerId: ENTITY_ID]

    onSkillEffect:
      side: client
      args: [skillId: UINT32, targetId: ENTITY_ID, effectData: BLOB]
```

#### Unity / C# 输出

// Generated by theseed-codegen — DO NOT EDIT

namespace Theseed.Unity.Entities
{
    public partial class Avatar : EntityBase
    {
        // --- 同步属性 ---
        public SyncProperty<string> name { get; }
        public SyncProperty<uint> level { get; }
        public InterpolatedProperty<float> hp { get; }
        public InterpolatedProperty<Vector3> position { get; }
        public InterpolatedProperty<Vector3> rotation { get; }
        public SyncProperty<float> mp { get; }

        // --- 事件回调（脚本层实现 partial 方法） ---
        partial void OnLevelUp(uint newLevel);
        partial void OnDamaged(float damage, ulong attackerId);
        partial void OnSkillEffect(uint skillId, ulong targetId, byte[] effectData);

        // --- 客户端调用服务器方法 ---
        public void attack(ulong targetId, uint skillId)
            => CallServer("attack", targetId, skillId);

        public void useItem(uint itemId, uint count)
            => CallServer("useItem", itemId, count);

        public void move(Vector3 position, Vector3 rotation)
            => CallServer("move", position, rotation);

        // --- 内部（SDK 使用） ---
        internal override void OnPropertySync(int propertyId, BundleReader reader)
        {
            switch ((AvatarProperty)propertyId)
            {
                case AvatarProperty.name:
                    name.Update(reader.ReadString());
                    break;
                case AvatarProperty.hp:
                    hp.SetTarget(reader.ReadFloat());
                    break;
                case AvatarProperty.position:
                    position.SetTarget(reader.ReadVector3());
                    break;
                // ... 自动生成
            }
        }

        internal override void OnMethodCall(int methodId, BundleReader reader)
        {
            switch ((AvatarMethod)methodId)
            {
                case AvatarMethod.onLevelUp:
                    OnLevelUp(reader.ReadUInt32());
                    break;
                case AvatarMethod.onDamaged:
                    OnDamaged(reader.ReadFloat(), reader.ReadUInt64());
                    break;
                // ... 自动生成
            }
        }

        // --- 插值更新（每帧调用） ---
        internal override void UpdateInterpolation(float deltaTime)
        {
            hp.Update(deltaTime);
            position.Update(deltaTime);
            rotation.Update(deltaTime);
        }
    }

    // 枚举自动生成
    internal enum AvatarProperty
    {
        name = 1,
        level = 2,
        hp = 3,
        position = 4,
        rotation = 5,
        mp = 6,
    }

    internal enum AvatarMethod
    {
        onLevelUp = 1,
        onDamaged = 2,
        onSkillEffect = 3,
        attack = 100,       // exposed 方法在 100+
        useItem = 101,
        move = 102,
    }
}

### 3.2 断线重连

```csharp
public class ReconnectManager {
    // 策略
    public ReconnectConfig Config { get; set; }

    // 流程
    // 1. 检测断线 → 进入重连状态
    // 2. 指数退避重试（1s, 2s, 4s, 8s, max 30s）
    // 3. 重连成功 → 恢复会话（服务器保持 entity 状态）
    // 4. 恢复失败 → 通知上层（显示"网络中断"界面）
    // 5. 超过最大重试次数 → 断开，通知上层

    // 服务器端配合：
    // - BaseApp 保持 entity 不销毁，等待 reconnect
    // - 超时（可配置，如 300s）后才销毁
    // - 重连时用 session token 恢复，不需要重新登录
}

public class ReconnectConfig {
    public bool enabled = true;
    public int maxRetries = 10;             // 最大重试次数
    public int initialDelayMs = 1000;       // 初始延迟
    public int maxDelayMs = 30000;          // 最大延迟
    public float backoffMultiplier = 2.0f;  // 退避倍数
    public int serverTimeoutMs = 300000;    // 服务器保持 entity 的超时
}
```

### 3.3 实体同步引擎

```csharp
public class EntitySyncEngine {
    // 实体生命周期
    public event Action<EntityBase> OnEntityEnter;      // 进入视野
    public event Action<EntityBase> OnEntityLeave;      // 离开视野
    public event Action<EntityBase> OnEntityUpdate;     // 属性更新

    // 玩家实体
    public EntityBase PlayerEntity { get; }             // 自己的玩家实体

    // 实体查询
    public EntityBase GetEntity(ulong entityId);
    public IEnumerable<T> GetEntitiesOfType<T>() where T : EntityBase;

    // 视野内实体数量
    public int VisibleEntityCount { get; }
}

// 插值属性
public class InterpolatedProperty<T> {
    public T current { get; }       // 当前插值后的值（用于渲染）
    public T target { get; }        // 目标值（服务器最新值）
    public T previous { get; }      // 上一个值

    public void SetTarget(T value);  // 收到服务器更新时调用
    public void Update(float dt);    // 每帧调用，向 target 插值
}
```

### 3.4 插值策略

```yaml
# 插值是游戏客户端体验的关键

属性类型          插值策略              说明
─────────────────────────────────────────────────────────
position         线性插值 + 预测       服务器位置 → 客户端平滑移动
rotation         球面插值 (SLERP)      旋转平滑
hp/mp            线性插值              血条平滑变化
level            不插值（直接跳变）    等级变化是离散事件
name             不插值               字符串不需要插值
velocity         不插值               用于客户端预测

高级策略：
  1. 客户端预测 + 服务器纠正
     - 客户端移动后立即预测结果
     - 服务器确认/纠正
     - 差异 > 阈值时回滚到服务器值

  2. 远近实体不同频率
     - detail_level=0: 每次更新都同步（附近实体）
     - detail_level=1: 降频同步（远处实体）
     - detail_level=2: 仅关键属性（很远）
```

public class TheseedManager : MonoBehaviour {
    public string serverHost = "127.0.0.1";
    public int serverPort = 20000;

    private TheseedClient client;

    async void Start() {
        client = new TheseedClient();

        // 注册实体创建工厂
        client.RegisterEntityFactory<Avatar>((id) => {
            // 自动实例化 Prefab 或让用户自定义

            var avatar = go.AddComponent<Avatar>();

            return avatar;

        await client.Login("account", "token");

        // 驱动 SDK tick（插值更新、消息处理）

## 6. 安全设计

### 6.1 Exposed 方法信任边界

```yaml
# 只有显式标记 exposed: true 的方法才能被客户端调用
# 代码生成器只为 exposed 方法生成 CallServer 函数
# 非 exposed 方法不会出现在客户端生成的代码中

methods:
  attack:
    exposed: true              # 客户端可调用

  onLevelUp:
    exposed: false             # 服务器推送，客户端不可调用
```

### 6.2 输入验证

```cpp
// 服务器端自动验证 exposed 方法的参数
// 1. 类型检查（消息反序列化时自动校验）
// 2. 范围检查（可选，在 def 中声明）
// 3. 权限检查（脚本层 onBeforeCall 钩子）

# YAML 中的验证规则
methods:
  attack:
    side: cell
    args:
      - name: targetId
        type: ENTITY_ID
        validate: "exists"          # 目标实体必须存在
      - name: skillId
        type: UINT32
        validate: "range(1, 1000)"  # 技能 ID 范围
    exposed: true
    rate_limit: 5/s                 # 每秒最多调用 5 次
```

## 7. 性能考量

### 7.1 消息编码

```
客户端协议设计原则：
1. 二进制编码（不是 JSON/Protobuf）
2. 变长整数（varint）减少带宽
3. 属性 ID 用整数，不是字符串
4. 位置/旋转用压缩格式（PackXZ/PackY）
5. 只同步变化了的属性（delta sync）
```

### 7.2 带宽优化

```
优化手段：
1. detail_level 分级同步
   - detail_level=0: 自己 + 附近实体（全量）
   - detail_level=1: 中距离（降频 + 位置低精度）
   - detail_level=2: 远距离（仅关键属性）

2. alias 机制
   - 首次发送包含 entity type string
   - 后续发送用 alias (uint8) 代替

3. 属性 delta
   - 只发送变化的属性
   - 用 bitmap 标记哪些属性变了

4. 消息合并
   - tick 内多个属性变更合并为一条消息
   - 同目标的消息合并发送
```

## 8. 与 KBEngine 的对比

| 能力 | KBEngine | theseed |
|------|---------|---------|
| **Unity** | 有插件，但维护不活跃 | 自动生成 + 编辑器集成 |
| **UE5** | 无官方支持 | Blueprint 全集成 |
| **Cocos** | 无官方支持 | TypeScript 自动生成 |
| **协议生成** | 无（手写客户端代码） | theseed-codegen 自动生成 |
| **属性插值** | 无内建 | InterpolatedProperty 开箱即用 |
| **断线重连** | 有限 | 自动重连 + 服务器保持 |
| **编辑器工具** | 无 | Entity Viewer + Network Profiler |
| **Exposed 安全** | 无自动验证 | 类型 + 范围 + 频率限制 |
| **带宽优化** | alias + detailLevel | 继承 + delta sync + 压缩 |
